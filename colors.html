Colours

Colour Palette

Colours plays a crucial role in software development, influencing both aesthetics and usability. A well-chosen colour palette enhances user experience by improving readability, guiding user interactions, and establishing visual hierarchy. In UI/UX design colours can evoke emotions, communicate brand identity, and provide accessibility for users with visual impairments. Consistent use of colours also helps users navigate interfaces intuitively, reducing cognitive load.

TailwindCSS offers a wonderful colour palette with plenty to choose from. Every colour in the palette includes 11 steps, with 50 being the lightest and 950 being the darkest.With such a versatile range of colours at your fingertips, you can create visually stunning, accessible, and intuitive designs. So go ahead—experiment, mix, and match to craft an interface that not only looks great but also enhances user experience!

Colour Utilities

Using colour utilities is very straight forward, for example, bg-teal for teal coloured background or have your text in orange by using text-orange-700 to set colour properties. To target the dark mode simply use dark variable, for example, dark:text-white-500.

  • bg-*

    The background color of an element

  • text-*

    The text color of an element

  • decoration-*

    The text decoration color of an element

  • border-*

    The border color of an element

  • outline-*

    The outline color of an element

  • shadow-*

    The color of box shadows

  • inset-shadow-*

    The color of inset box shadows

  • ring-*

    The color of ring shadows

  • inset-ring-*

    The color of inset ring shadows

  • accent-*

    The accent color of form controls

  • caret-*

    The caret color in form controls

  • fill-*

    The fill color of SVG elements

  • stroke-*

    The stroke color of SVG elements

Colour Opacity

You can change the opacity of a chosen colour using syntax such as bg-emerald/50, in this case '50' sets the alpha channel of the colour to 50%:

As a handy option this syntax also supports arbitrary values and the CSS variable shorthand: 

  • bg-white-300/[65.75%]
  • bg-teal-700/(--custom-teal)

OKLCH

As TailwindCSS uses the OKLCH color model - a perceptually uniform alternative to traditional color models like RGB and HEX, let's look into what exactly is it.

What is OKLCH?

OKLCH stands for OKLab (Lightness, Chroma, and Hue), which is designed to represent colours in a way that aligns better with human perception. The OKLCH colour space was created to offer consistent perceptual differences. In simpler terms, colours that are close to each other in OKLCH will appear equally close to the human eye, making it more intuitive for designers and more accurate for colour manipulations.

OKLCH Components

  • L: Lightness (0–1 scale, where 0 is black and 1 is full white)
  • C: Chroma (how intense the color is, 0 means gray)
  • H: Hue (the color itself, in a circular scale 0–360°)

For example, oklch(0.971 0.013 17.38) describes a color with:

  • Lightness = 0.971 (very light)
  • Chroma = 0.013 (very muted, almost gray)
  • Hue = 17.38° (which corresponds to a very light reddish tone)

Check out this OKLCH Color Picker & Converter, an amazing tool to pick the colour you want.

How is OKLCH Different from HEX?

  • Perceptual Uniformity: The most significant advantage of OKLCH over HEX is its better perceptual uniformity. When you change a value in OKLCH (for example, adjust the lightness or chroma), the visual difference will be consistent and predictable across all displays, unlike RGB/HEX where the colour can shift unpredictably due to device differences.
  • More Intuitive for Designers: OKLCH is closer to how humans perceive colour, making it easier to create visually harmonious colour palettes. HEX colors are not as perceptually uniform, meaning that small changes in the colour values can produce noticeable changes in hue or brightness that may not feel "natural."
  • Better for Colour Manipulation: With OKLCH, you can manipulate colours based on how people perceive them. You can adjust lightness or chroma without distorting the underlying hue, which is harder to achieve with RGB or HEX.

Advantages of Using OKLCH

  • More predictable color adjustments: Changing the values of the lightness, chroma, or hue components will have a more uniform visual effect.
  • Improved accessibility: Colours in OKLCH can be better fine-tuned for contrast, making them more accessible to people with color vision deficiencies.
  • Consistency across devices: OKLCH handles perceptual differences better than RGB/HEX across different screens.

OKLCH vs HEX

HEX is still widely used because of its compatibility across browsers and tools, and will certaintly stay around. It's simple, well-understood, and easy to use in most cases. OKLCH on the other hand can be better if you're working on more advanced color manipulations where perceptual accuracy is essential, such as advanced design systems, data visualization, or high-end UI/UX projects. If you're manipulating lightness and chroma while maintaining a consistent appearance, OKLCH is the better choice.
